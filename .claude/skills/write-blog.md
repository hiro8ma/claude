# ブログ記事の作成・編集

$ARGUMENTS

## ルール

### 文体
- 敬語（です・ます調）で統一
- カジュアルな茶目っ気はOK（笑、絵文字等）
- コロン（:）を見出しにつけない

### 段落
- 1文1行を基本にする
- 意味のまとまりで空行を入れて段落分け

### 構成
- 各セクションの冒頭に導入文を丁寧に書く
- 前のセクションからの繋がりを意識する
- テーブル・blockquote・boldで視覚的に見せる
- ASCII図よりMarkdownネイティブの記法を使う

### 表現
- 「書籍」→「文献・記事」（著作権配慮）
- Claude Code機能名は公式の英語表記（Remote Control、Agent Teams、`/voice`等）
- 具体的な技術詳細は書きすぎない
- 「現時点」「執筆時点（年月）」を明記
- 項目には「等」をつけて余白を残す

### 絵文字
- 見出しに絵文字をつける
- 文末の感情表現に使う
- ドメインや役割にアイコンとして使う

### 公開配慮
- Public リポジトリにはリンクを貼る
- Private リポジトリは具体名を出さない
- センシティブな内容は書かない

### タイトル
- 既存記事と被らないか事前にリサーチ
- 技術キーワードを含めつつキャッチーに
